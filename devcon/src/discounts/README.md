# `src/discounts`

Discount-eligibility lists consumed by the discount validator + claim endpoints
(`src/pages/api/discounts/validate/[id].ts`, `claim/[id].ts`, `claim-wallet.ts`).
Each entry is a lowercased wallet address or GitHub username.

These are generated in the sibling **discounts** repo (`../discounts`) and pulled in here.
Edit the source/script there, not the JSON by hand.

| File | Discount tier | Generated by (discounts repo) | Regenerate (from `../discounts`) |
|---|---|---|---|
| `core-devs.json` | Core Devs / Protocol Guild (free) | `src/core.ts` -> `outputs/core-devs.json` (merges `inputs/the-merge-contributors.json` + `inputs/protocol-guild-2026.json`, resolves ENS) | `bun run core` |
| `oss-contributors.json` | OSS Contributors (50%, now folded into the builder discount) | `src/github.ts` -> `outputs/oss-contributors.json` (scans `efdevcon`/`ethereum` orgs + execution clients; needs `GITHUB_TOKEN`) | `bun run github` |
| `pg-projects.json` | Public Good Projects (50%) | `src/pg.ts` -> `outputs/pg-projects.json` (merges Gitcoin / Giveth / Octant / Optimism source lists) | `bun run pg` |
| `past-attendees.json` | Past Attendees / POAP (10%) | `src/poaps.ts` -> `outputs/poap-past-attendees.json` (merges `inputs/POAP_drop_*.csv` from `bun run poap-fetch`) | `bun run poap` |

## Syncing

The files here are pulled from `../discounts/outputs/` by a single devcon-owned script,
`src/scripts/sync-discount-data.ts` (the from/to mapping lives there). Workflow:

```bash
# 1. regenerate whichever list you changed, in the discounts repo
cd ../discounts && bun run core   # or: github | pg | poap

# 2. pull the fresh outputs into this repo
cd ../devcon && pnpm sync:discount-data

# (optional) report drift without writing — useful in CI
pnpm sync:discount-data --check
```

Notes:

- The sync **overwrites** these files from the discounts outputs (the outputs are the source
  of truth), so always regenerate in `../discounts` first. Don't hand-edit these JSONs.
- The poap output is renamed on the way in: `poap-past-attendees.json` -> `past-attendees.json`
  (the name the validator imports).
- `pnpm sync:discount-data` also refreshes the builder scoring lists in `src/data/` (OSO repos).
  See `src/data/README.md`.
